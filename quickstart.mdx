---
title: "Get Blocklet Server Running in Minutes"
description: "Choose between Docker (recommended) and npm to get Blocklet Server running locally, then install your first Blocklet via the dashboard or CLI."
---

Blocklet Server supports two installation paths: Docker and npm. Docker is recommended for production and for getting started quickly, since it bundles all dependencies including nginx. The npm path installs the `@blocklet/cli` package and is useful when you want tighter integration with an existing Node.js environment.

## System requirements

Before you start, confirm your environment meets the minimum requirements.

| Requirement | Minimum | Recommended |
|-------------|---------|-------------|
| Node.js | 20.x | 22.x or later |
| Memory | 2 GB RAM | 4 GB RAM or more |
| Disk space | 10 GB | 50 GB or more |
| OS | Linux, macOS, Windows (via Docker) | Linux (Ubuntu 20.04+) |

<Tip>
  If you are on Windows, use the Docker path — Blocklet Server requires Linux system features that are not available natively on Windows. WSL2 works as an alternative for the npm path.
</Tip>

## Option 1: Docker (recommended)

<Steps>
  <Step title="Pull the image">
    Pull the official Blocklet Server image from Docker Hub:

    ```bash
    docker pull arcblock/blocklet-server
    ```
  </Step>
  <Step title="Run the container">
    Start the container with port mappings for HTTP and HTTPS, and a volume mount so your data persists across restarts:

    ```bash
    docker run -d \
      -p 80:80 \
      -p 443:443 \
      -v ~/blocklet-data:/data \
      arcblock/blocklet-server
    ```

    The `-v ~/blocklet-data:/data` flag mounts a local directory to `/data` inside the container. All server state, installed Blocklets, and configuration are stored there.
  </Step>
  <Step title="Access the dashboard">
    Open [http://localhost](http://localhost) in your browser. The first-run wizard will guide you through setting your server domain, creating your admin account, and generating your server DID.
  </Step>
  <Step title="Install your first Blocklet">
    From the dashboard, click **Store** to browse available Blocklets and install one with a single click. Or install directly from the CLI:

    ```bash
    blocklet install https://store.blocklet.dev/blocklets/z8ia29UsENBg6tLZUKi2HABj38Cw1LmHZo8e
    ```
  </Step>
</Steps>

## Option 2: npm

<Steps>
  <Step title="Install the CLI">
    Install `@blocklet/cli` globally using npm:

    ```bash
    npm install -g @blocklet/cli
    ```

    This installs the `blocklet` command on your PATH. Confirm installation with:

    ```bash
    blocklet --version
    ```
  </Step>
  <Step title="Start the server">
    Run the server with the `-a` flag to automatically initialize and start:

    ```bash
    blocklet server start -a
    ```

    The server prints its dashboard URL on startup. By default this is [http://localhost](http://localhost).
  </Step>
  <Step title="Access the dashboard">
    Open the URL printed in your terminal — typically [http://localhost](http://localhost). Complete the initial setup wizard to configure your server.
  </Step>
  <Step title="Install your first Blocklet">
    From the dashboard, click **Store** to browse and install Blocklets. You can also install from the CLI:

    ```bash
    blocklet install https://store.blocklet.dev/blocklets/z8ia29UsENBg6tLZUKi2HABj38Cw1LmHZo8e
    ```
  </Step>
</Steps>

<Note>
  The npm path requires Node.js 20.x or later and nginx 1.18+ installed separately on your system. If you do not have nginx, use the Docker path instead.
</Note>
